01 · PROBLEM & GOAL

사람의 실수가 아니라 구조의 문제

업무 지식이 개인에게 머물고, 기존 파일 복사가 사실상의 표준이 되며, 검토가 최종금액을 훑어보는 수준으로 형식화된 것이 반복 오류의 원인입니다.

PROBLEM 01

입찰공고문 날짜·조건 오류

문서 여러 곳에 흩어진 날짜를 일부만 수정하여 공고 취소·재공고와 일정 지연이 발생합니다.

PROBLEM 02

원가계산 수식·제비율 오류

전임자 엑셀의 행·열을 수정하며 셀 참조가 깨지고, 반기별 제비율 개정을 놓쳐 예산·감사 리스크로 이어집니다.

PROBLEM 03

견적서 전사·부가세 오류

이미지·PDF 견적서를 눈으로 옮겨 적는 과정에서 금액 오기입과 부가세 이중 반영이 발생합니다.

ROOT CAUSE

지식의 개인화 + 복사·수정 관행

최신 기준과 판단 근거를 시스템이 보유하지 않아 담당자 교체나 기준 개정 때 정보가 낡거나 유실됩니다.

DESIGN PRINCIPLE

자동 검증, 사람의 최종 판단

반복 대조·계산·최신화는 자동화하되 법령 적용과 예외승인은 담당자가 근거를 확인하고 확정합니다.

“한 번 입력한 데이터가 원가검증과 공고문 작성까지 이어지고, 시스템이 누락과 불일치를 먼저 찾아내는 업무 흐름”MVP 한 문장 정의를 화면·데이터·승인 흐름으로 재구성
